# Task Completion Checklist

When a programming task is completed, ensure the following steps are taken:

1. **Linting**: Run `make lint` to check for style issues and errors.
2. **Formatting**: Run `make format` to normalize code style.
3. **Testing**: Run `make test` to verify changes didn't break existing functionality.
4. **Documentation**: Update docstrings and README if applicable.
5. **Git**: Add, commit, and push changes if directed by the task.
